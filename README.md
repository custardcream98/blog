## Commit Message Convention

**2022/10/05부터 적용**

- `Post` : 포스팅 업로드
- `Typo` : 문서 오타 수정
- `Change` : 문서 내용 수정
- `Chore` : minor change
- `Refactor` : 리팩토링
- `Optimize` : 최적화
- `Bugfix` : 버그 수정
- `Feature` : 기능 추가
- `Todo` : Todo 추가, Fixme 추가 등

* 가능하면 간결한 한글로, Commit Message Prefix와 겹치는 내용 없이
  - 잘못된 예) `Typo: 자바스크립트의 변수들 오타 수정`
  - 올바른 예) `Typo: 자바스크립트의 변수들`
